# Security Policy

## Reporting a vulnerability

Please do not open a public issue for sensitive security reports.

Send a private report to the maintainer through GitHub security advisories if
available, or email the maintainer listed on the GitHub profile.

Include:

- A clear description of the vulnerability.
- Steps to reproduce.
- Affected versions or commits.
- Any suggested fix or mitigation.

The maintainer will acknowledge valid reports as quickly as possible and
coordinate a fix before public disclosure.
